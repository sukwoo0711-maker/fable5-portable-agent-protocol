# Pre-Final Review

## Diff Audit

- [ ] Every changed file serves the request.
- [ ] No unrelated formatting or refactor churn.
- [ ] No user work overwritten.
- [ ] Temporary or debug artifacts removed.

## Verification Audit

- [ ] Each success claim maps to observed evidence.
- [ ] Failed or skipped checks are named.
- [ ] Static-only conclusions are labeled.

## Safety Audit

- [ ] No destructive, external, production, or device action without approval.
- [ ] Generated or vendor files were handled through source or disclosed.

## Reporter Audit

- [ ] Outcome is short and concrete.
- [ ] Evidence and missing verification are both reported.
- [ ] Last paragraph is not an unexecuted plan, promise, or non-blocking question.
